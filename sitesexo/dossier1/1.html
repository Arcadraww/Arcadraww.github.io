Quelques remarques

  1. Sauf cas exceptionnels, la mise en forme dans le document HTML source n'est pas prise en compte, en particulier les retours à la ligne.
  2. Le cas de la balise <img> est particulier, celle-ci ne comporte pas de contenu et n'a donc pas de balise fermante.
  3. Certains caractères spéciaux sont codés par une entité dans le code source. Ces entités commencent pas le symbole "&" et se terminent par ";". Les entités peuvent se référencer par un nom ou par le code hexadécimal du caractère en UTF-8, ce code est alors précédé de #x.

    En XHTML5, seules quelques entités nommées sont reconnues, notamment &lt; et &gt; pour les caractères "<" et ">". Pour les autres il faut utiliser le code UTF-8.

    Voici quelques autres exemples d'entités avec leur nom HTML et leur code UTF-8 :

    • &nbsp; ou &#xA0; pour un caractère blanc insécable " ", nécessaire pour respecter les règles typographiques, par exemple avant un ":"
    • &laquo; ou &#xAB; et &#xBB;/&#187; pour les caractères "«" et "»"
    • &euro; ou &#x20AC; pour le caractère "€"
    • &frac12; ou &#xBD; pour le caractère "½"
    • etc.

    Il en existe beaucoup d'autres, vous pouvez en trouver une liste complète ici par exemple

  4. Les commentaires s'écrivent dans le code source entre les balises spéciales <!-- et -->. Cette partie du document est ignorée par le navigateur.
